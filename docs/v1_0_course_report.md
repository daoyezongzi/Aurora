# Aurora v1.0 课程报告草稿

## 1. 课题名称
基于时序特征工程与稳健评估的股票次日涨跌二分类研究（Aurora）

## 2. 研究背景与问题定义
股票涨跌预测在金融信息处理课程中是典型二分类问题。相比只追求单次高分，本项目强调：
- 模型流程完整可复现
- 指标可解释
- 支持版本迭代与工程化迁移

任务定义：
- 输入：某股票历史日线及派生特征
- 输出：次日上涨概率与分类结果（`target_up=1{close_{t+1}>close_t}`）

## 3. 数据来源与处理流程
- 数据源：Tushare 日线数据（以 `002372.SZ` 为示例）。
- 处理流程：缺失处理 -> 特征构建 -> 时间切分 -> 标准化。
- 切分策略：按时间顺序 `70/15/15`，避免未来信息泄漏。

## 4. 方法设计
### 4.1 基线与对比（v0.2）
- LSTM
- Logistic Regression
- MLP

### 4.2 主线方案（v0.3）
- 使用 MLP 作为主模型。
- 引入 rolling 验证评估稳定性。
- 阈值调优目标使用 f0.5，并加入 precision/recall 门槛约束。
- 输出错误分析，定位误报/漏报结构。

### 4.3 特征迁移验证（v0.4）
- 同一套特征分别用 pandas 与 SQL 风格窗口实现。
- 对齐特征一致性 + 对齐训练结果一致性。

## 5. 实验结果
### 5.1 多模型对比（v0.2）
- MLP 在本任务下优于 LSTM 与 LogReg（`f1=0.6061`）。

### 5.2 稳健主线结果（v3 final）
来自 `outputs/v3/final/metrics.json`：
- accuracy = 0.5260
- f1 = 0.5882
- precision = 0.5200
- recall = 0.6771
- roc_auc = 0.5474

### 5.3 迁移一致性结果（v4 compare）
来自 `outputs/v4/compare_metrics.csv`：
- pandas 与 SQL 的关键指标差值均为 0.0。
- 说明迁移实现达到功能等价与结果可复现。

## 6. 分析与讨论
- 项目收益：不仅给出分数，还给出稳定性、阈值策略和错误结构。
- 当前不足：
  - 指标绝对值仍有提升空间；
  - 单股票样例泛化性有限；
  - 尚未引入交易成本与真实策略回测。

## 7. 结论
本项目完成了课程要求的完整机器学习流程，并在工程层面实现了：
- 版本化实验管理
- 稳健评估机制
- 特征工程迁移一致性验证

因此，Aurora v1.0 既可用于课程答辩，也具备后续对接业务系统（如 TradeEye）的基础。
