# Aurora v1.0 答辩问答清单

## Q1：你这个项目到底解决什么问题？
A：解决“根据历史日线特征预测次日涨跌”的二分类问题。重点不是单次高分，而是完整、可复现、可解释的机器学习流程。

## Q2：为什么按时间切分，不随机切分？
A：时序数据必须避免未来信息泄漏。时间切分更符合真实交易场景。

## Q3：为什么最终主线是 MLP，不是 LSTM？
A：在当前数据和特征条件下，v0.2 对比结果中 MLP 的 F1 更优且稳定性更好，工程复杂度也更低。

## Q4：为什么要做 rolling 验证？
A：单次切分偶然性强。rolling 可以观察跨时间窗口稳定性，更能说明模型是否可用。

## Q5：阈值为什么不是固定 0.5？
A：0.5 不是业务最优。通过阈值调优可以在误报和漏报之间做可解释的权衡。

## Q6：v0.4 的价值是什么？
A：验证 pandas 与 SQL 风格特征工程迁移一致性。对比结果显示关键指标差值为 0，说明迁移等价成功。

## Q7：现在这个版本能交作业吗？
A：能。v1.0 已具备完整流程、指标产物、错误分析和文档闭环，满足课程答辩基本要求。

## Q8：后续如何对接 TradeEye？
A：先固定输出数据契约，再做脚本级联调，最后服务化接入。具体步骤见 `docs/v1_0_tradeeye_integration.md`。
