# Aurora v1.0 PPT 思路（答辩可直接用）

## 1. 建议页数
- 5~7 页主内容 + 1 页备用问答。

## 2. 逐页结构
### 第 1 页：题目与目标
- 题目：股票次日涨跌二分类（Aurora）
- 目标：做出可复现、可解释、可迁移的 ML 流程

### 第 2 页：数据与任务定义
- 数据来源、时间范围、标签定义
- 为什么按时间切分而不是随机切分

### 第 3 页：方法演进
- v0.1：最小闭环
- v0.2：模型对比
- v0.3：rolling + 阈值 + 错误分析
- v0.4：pandas -> SQL 特征迁移

### 第 4 页：核心结果
- v3 final 指标（f1/precision/recall/roc_auc）
- 错误结构（FP/FN）

### 第 5 页：迁移对比结果
- v4 compare：关键指标差值为 0
- 结论：迁移等价成功

### 第 6 页：工程化与交付
- 一键脚本、版本化输出、文档闭环
- 为什么这对课程答辩和后续迭代都重要

### 第 7 页：总结与下一步
- 当前结论
- 后续对接 TradeEye 的计划

## 3. 2 分钟讲稿骨架
1. 问题与目标（20 秒）
2. 方法演进（35 秒）
3. 结果与结论（45 秒）
4. 迁移价值与下一步（20 秒）

## 4. 老师高频追问准备
- 为什么不用随机切分？
- 为什么 MLP 是主线？
- 阈值为什么不是固定 0.5？
- v4 的“差值为 0”说明了什么？
- 项目如何落地到业务系统？
